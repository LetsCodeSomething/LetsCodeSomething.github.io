ДОБРО ПОЖАЛОВАТЬ!
Abyss Walker (Путник Бездны) - это простой генератор фракталов, написанный на языке JavaScript. Abyss Walker позволяет создавать изображения множества Мандельброта и сохранять их на Вашем компьютере.
Для тех, кто хорошо знаком с фракталами: данный генератор не использует длинную арифметику и оптимизацию алгоритма с помощью «perturbation theory».
МНОЖЕСТВО МАНДЕЛЬБРОТА
Множество Мандельброта - это множество точек, лежащих на комплексной плоскости, для которых рекуррентное соотношение zn+1 = zn2 + c при z0 = 0 (назовём её формулой *) задаёт ограниченную последовательность. Координаты каждой точки можно записать в виде x + iy, где x - действительная часть комплексного числа, а y - мнимая.
Множество Мандельброта было впервые описано в 1905 году французским математиком Пьером Фату. Фату обнаружил, что орбита для начального условия (набор координат точки, получаемый в результате выполнения формулы *) z0 = 0 показывает достаточно сложное и интересное поведение. В те времена компьютеров ещё не было, и Фату, конечно, не мог построить орбиты всех точек плоскости, ему приходилось всё делать вручную. Основываясь на своих расчётах, он доказал, что орбита точки, лежащей на расстоянии больше 2 от начала координат, всегда уходит в бесконечность.
Профессор Бенуа Мандельброт был первым, кто использовал компьютер для визуализации множества. Фракталы были описаны Мандельбротом в 1975 году в его книге «Les Objets Fractals: Forme, Hasard et Dimension» («Фрактальные объекты: форма, случайность и размерность»). В этой книге Мандельброт впервые использовал термин «фрактал» для обозначения математического феномена, демонстрирующего столь непредсказуемое и удивительное поведение. Множество Мандельброта назвали по имени его исследователя.
Изображения множества Мандельброта создаются с помощью итерации (циклического выполнения) формулы * для каждой точки комплексной плоскости. Параметр c, по сути, представляет собой координаты точки на плоскости, для который выполняются итерации. В идеале формула * должна выполняться бесконечное количество раз, но при таком подходе изображение будет невозможно получить. Поэтому берётся некоторое значение n, обозначающее предел количества итераций. Процесс циклического выполнения формулы * останавливается, если количество выполненных итераций равно n или если модуль |zn| получился равным или больше 2.
В зависимости от результата циклического выполнения формулы * и выбранного способа раскрашивания точек точкам присваиваются различные цвета. При использовании обычной палитры точка закрашивается чёрным цветом (точка принадлежит множеству), если в конце итераций модуль |zn| меньше 2, иначе точка закрашивается белым цветом. Кроме математической палитры существуют и другие. Одной из самых популярных является палитра, присваивающая цвета точкам на основе количества итераций, через которое точка удаляется в «бесконечность» (Рисунок 2). Палитра циклическая, поэтому бояться того, что не хватит цветов, не нужно.
Рисунок 2. Раскрашивание внешней области по скорости удаления точек
Можно заметить, что при использовании такой палитры видно границы между областями разного цвета. Для устранения этих границ был придуман метод сглаживания цветовых переходов, основанный на расстоянии, на которое точка удаляется от круга с центром в точке 0 i0 и радиусом 2 (Рисунок 3).
